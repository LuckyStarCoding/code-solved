## ğŸ“ Task Description
Write a function that takes a list of hours (Monday to Sunday) and returns the total weekly salary based on these rules:
- **Standard Rate:** 10 per hour for the first 8 hours.
- **Overtime Rate:** 15 per hour for hours beyond 8.
- **Weekends:** Employer pays double the usual rate for all hours.

    This approach uses four distinct mathematical helpers:
    1. **sn:** Weekday Base (Hours * 10)
    2. **so:** Weekday Overtime (First 8 hours at 80 + remaining hours at 15)
    3. **wn:** Weekend Base (Double the normal rate)
    4. **wo:** Weekend Overtime (Double the weekday overtime rate)

    

    > **Mentor Tip:** Defining your rules as small functions (`sn`, `so`, etc.) is a pattern often seen in "Declarative Programming." It makes the core loop very easy to read because it looks like a simple decision tree.

    ```javascript
    function weeklySalary(h) {
      let total = 0;

      // Define calculation rules as helper functions
      let sn = (t) => t * 10;                  // Standard Normal: $10/hr
      let so = (t) => (t - 8) * 15 + 80;       // Standard Overtime: $15/hr after 8hrs ($80)
      let wn = (t) => t * 10 * 2;              // Weekend Normal: Double rate ($20/hr)
      let wo = (t) => (t - 8) * 15 * 2 + 160;  // Weekend Overtime: Double overtime rate

      for (let i = 0; i < h.length; i++) {
        // Mon (0) to Fri (4) are Weekdays
        if (i <= 4) {
          total += h[i] <= 8 ? sn(h[i]) : so(h[i]);
        } 
        // Sat (5) and Sun (6) are Weekends
        else {
          total += h[i] <= 8 ? wn(h[i]) : wo(h[i]);
        }
      }
      
      return total;
    }
    ```

  <TabItem value="notes" label="Notes & Security">
    ### ğŸ›¡ï¸ Guardrails
    - **Memory Efficiency:** By using a `total` accumulator instead of an intermediate array, we keep the space complexity at O(1).
    - **Logic Boundaries:** Using `i <= 4` strictly targets the first 5 elements. Always ensure your input array represents a full 7-day week to prevent logic errors.
    - **Security Hint:** In financial applications, floating-point math can lead to "penny errors." Since these are whole numbers, it's safe, but always be cautious if rates include decimals.
