# Master the Two Pointers Pattern: Solving Two Sum II

The **Two Pointers** technique is one of the most efficient ways to search for pairs in a sorted dataset. Instead of a nested loop that results in O(n^2) time complexity, this approach allows us to find the answer in a single passâ€”O(n).

## The Challenge: Two Sum II (Sorted Input)

Given an array of integers `numbers` that is already **sorted in non-decreasing order**, find two numbers such that they add up to a specific `target` number.

### Constraints:

- **Time Complexity:** O(n)
- **Space Complexity:** (1) (Constant space)
- **Output:** The result should be **1-indexed** (e.g., the first element is at index 1).

      ### The Visual Logic

      Instead of checking every possible pair, we place one pointer at the **start** and one at the **end**.

      - **Sum < Target:** Move the `start` pointer right (`s++`).
      - **Sum > Target:** Move the `end` pointer left (`e--`).

/**
 * Two Sum II - Input Array Is Sorted
 * Pattern: Two Pointers
 */
let find = (arr, t) => {
    // SECURITY/ROBUSTNESS: Handle edge cases where input is not an array or too short
    if (!Array.isArray(arr) || arr.length < 2) return [];

    let s = 0;             // Start pointer
    let e = arr.length - 1; // End pointer

    // Condition s < e ensures we don't use the same element twice
    while (s < e) {
        let a = arr[s] + arr[e]; // Current sum

        if (a === t) {
            return [s + 1, e + 1]; // Return 1-based indices
        } else if (a > t) {
            e -= 1; // Sum too large, move the right pointer left to decrease it
        } else {
            s += 1; // Sum too small, move the left pointer right to increase it
        }
    }

    // Explicitly return something if no pair is found (good practice)
    return [-1, -1]; 
}
