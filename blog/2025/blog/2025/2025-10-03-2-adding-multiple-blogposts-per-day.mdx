Ever been on a serious writing streak? I sure have! I ran into a peculiar little snag recently when I needed to publish **two separate blog posts on the same calendar day** in my Docusaurus site.

Docusaurus is awesome, but it's *very* strict about using the date in your filename (`YYYY-MM-DD-...`) for post ordering. If two posts share the exact same date, the system gets confused about which one is "first."

## ğŸ“… Method 1: The Filename Identifier (The Quick Fix)

The simplest and most robust way to solve this is to add a small, **unique identifier** right after the date in your file name. This makes the filename unique while still keeping the primary date correct.

### Step-by-Step Filename Structure

By default, Docusaurus looks for: `YYYY-MM-DD-your-slug.md`.

To make it unique, we just sneak a sequential number (or letter) in there:

`{YYYY-MM-DD-X-your-slug.md}`

Where **`X`** is a number (1, 2, 3...) or a letter (A, B, C...) that dictates the order on that day.

| Filename | Publication Date | Order on Blog Roll |
| :--- | :--- | :--- |
| `2025-10-02-1-morning-coffee-thoughts.md` | October 2, 2025 | 1st Post |
| `2025-10-02-2-afternoon-coding-tips.md` | October 2, 2025 | 2nd Post |
| `2025-10-02-3-late-night-ramblings.md` | October 2, 2025 | 3rd Post |

If you're like me and prefer to keep your filenames super clean, you can completely ignore the need for the sequential number in the file name and instead rely on the `date` field in your post's front matter.

If Docusaurus finds a `date` field in the front matter, it uses that instead of inferring the date from the filename! This gives you **explicit chronological control**.

### Step-by-Step Front Matter Structure

You need to use a precise **ISO 8601** string format, which includes the time and timezone (e.g., `YYYY-MM-DDTHH:mm:ssZ`).
